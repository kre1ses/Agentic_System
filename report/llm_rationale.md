# Обоснование выбора LLM и архитектуры агентов

## 1. Принципы выбора

При назначении модели каждому агенту учитывались три критерия:

| Критерий | Описание |
|----------|----------|
| **Сложность задачи** | Требует ли агент цепочечного рассуждения (Chain-of-Thought), структурированного JSON-вывода или достаточно детерминированного tool-routing |
| **Пропускная способность** | Как часто агент вызывается: разовые задачи (Planner) vs последовательные вызовы инструментов (Explorer, Engineer, Builder) |
| **Стоимость / доступность** | Бесплатные open-source модели предпочтительны; платные API — только там, где качество критично |

---

## 2. Поддерживаемые провайдеры

| Провайдер | Переменная окружения | Плюсы | Минусы |
|-----------|---------------------|-------|--------|
| **Anthropic** | `ANTHROPIC_API_KEY` | Лучшее качество рассуждений, надёжный tool use, большое контекстное окно | Платно, нет бесплатного tier |
| **OpenRouter** | `OPENROUTER_API_KEY` | Бесплатный tier, OpenAI-совместимый API, доступ к Llama/Mistral/Qwen | Free модели жёстко rate-limit'ятся, не все поддерживают tool use |
| **VseGPT** | `VSEGPT_API_KEY` | Российский прокси, доступ к o3-mini и другим моделям | Платно, зависимость от зарубежных провайдеров |
| **HuggingFace** | `HF_TOKEN` | Прямой доступ к open-source весам, бесплатно для публичных моделей | Ограниченное tool calling, нестабильность latency |

Провайдер **определяется автоматически** по наличию ключей в окружении (приоритет: Anthropic → OpenRouter → VseGPT → HuggingFace).

Агент `ValidationAgent` запускается **до** обращения к LLM-провайдеру: большая часть его работы — чистый pandas. LLM вызывается только один раз и только если найдены подозрения на leakage.

---

## 3. Выбор модели по агенту

### ValidationAgent — быстрая модель (преимущественно rule-based)

**Задача**: входной контроль данных до запуска LLM-агентов. Проверяет файлы, схему train/test, корректность target, пропуски, ID-колонки, константные признаки, дубликаты строк и риски утечки target. Работает на чистом pandas; LLM вызывается только один раз — для краткого summary найденных рисков утечки.

Требования к модели минимальны: нужно прочитать JSON с findings и написать 3–5 предложений.

| Провайдер | Модель | Обоснование |
|-----------|--------|-------------|
| Anthropic | `claude-haiku-4-5-20251001` | Самая быстрая модель; summary требует минимальных рассуждений |
| OpenRouter | `mistralai/mistral-small-3.1-24b-instruct:free` | 24B достаточно для суммаризации |
| VseGPT | `mistralai/mistral-small-3.1-24b-instruct` | Та же модель без ограничений free tier |
| HuggingFace | `meta-llama/Llama-3.1-8B-Instruct` | 8B; задача тривиальная для любой инструктируемой модели |

**Ключевые выходы агента**, передаваемые downstream:
- `drop_candidates` — ID-колонки, константные и 100%-пустые признаки → Engineer использует как обязательный список к удалению
- `leakage_suspects` — признаки с высокой корреляцией к target или подозрительными именами → Engineer исключает их из обучения
- `missingness_profile` — доля пропусков по каждому признаку → Engineer выбирает стратегию импутации
- `feature_type_map` — эвристические типы признаков → инициализирует решения Engineer
- `can_proceed` — если `False`, пайплайн останавливается немедленно с диагностикой

---

### PlannerAgent — крупная / reasoning-модель

**Задача**: декомпозиция задачи на подзадачи, построение графа зависимостей, учёт ограничений (утечка данных, порядок этапов). Требуется структурированный JSON-вывод и многошаговое рассуждение.

| Провайдер | Модель | Обоснование |
|-----------|--------|-------------|
| Anthropic | `claude-sonnet-4-6` | Лучший баланс качества и скорости; надёжный JSON-вывод |
| OpenRouter | `meta-llama/llama-3.3-70b-instruct:free` | 70B параметров; Llama 3.3 обучен следовать сложным инструкциям |
| VseGPT | `openai/o3-mini` | Специально оптимизирован для structured reasoning |
| HuggingFace | `Qwen/Qwen2.5-72B-Instruct` | 72B; Qwen-2.5 лидирует на reasoning-бенчмарках среди open-source |

---

### ExplorerAgent — быстрая tool-calling модель

**Задача**: последовательный вызов 6–7 EDA-инструментов. Каждый вызов детерминирован — инструмент возвращает данные, модель форматирует результат. Глубокого рассуждения не требуется, важна надёжность function calling.

| Провайдер | Модель | Обоснование |
|-----------|--------|-------------|
| Anthropic | `claude-haiku-4-5-20251001` | Самая быстрая модель Anthropic; надёжный tool use при минимальной задержке |
| OpenRouter | `mistralai/mistral-small-3.1-24b-instruct:free` | 24B; Mistral хорошо обучен на function calling |
| VseGPT | `mistralai/mistral-small-3.1-24b-instruct` | Та же модель без ограничений free tier |
| HuggingFace | `meta-llama/Llama-3.1-8B-Instruct` | 8B достаточно для детерминированного tool routing |

---

### EngineerAgent — быстрая tool-calling модель

**Задача**: анализ EDA-отчёта и принятие решений по признакам (что дропать, как кодировать, какие трансформации применить). Задача структурированная — небольшая модель с хорошим instruction-following справляется.

*Модели идентичны ExplorerAgent* — одинаковые требования к скорости и надёжности tool use.

---

### BuilderAgent — быстрая tool-calling модель

**Задача**: вызов `compare_models`, `train_and_evaluate`, `feature_importance`; интерпретация метрик. Выбор модели детерминирован метриками (минимальный MSE) — нужно корректно прочитать числа.

*Модели идентичны ExplorerAgent.*

---

### CriticAgent — крупная / reasoning-модель

**Задача**: рецензирование работы других агентов. Требуется:
- обнаружение тонких проблем (data leakage, overfitting, неверная стратегия импутации);
- разграничение major / minor ошибок;
- структурированный JSON-вывод: `{"approved": bool, "severity": str, "issues": [...], "suggestions": [...]}`.

Высокие требования к рассуждению → самая мощная доступная модель.

| Провайдер | Модель | Обоснование |
|-----------|--------|-------------|
| Anthropic | `claude-sonnet-4-6` | Лучшая критическая аналитика в линейке Claude |
| OpenRouter | `meta-llama/llama-3.3-70b-instruct:free` | 70B; Llama 3.3 показывает хорошие результаты на задачах peer review |
| VseGPT | `openai/o3-mini` | Reasoning-оптимизированная модель OpenAI |
| HuggingFace | `Qwen/Qwen2.5-72B-Instruct` | 72B; сильные результаты на аналитических задачах |

---

### CoordinatorAgent — крупная / reasoning-модель

**Задача**: оркестрация всего пайплайна, интерпретация результатов фаз, принятие решений о повторных итерациях. Координатор видит весь контекст → нужна модель с большим окном контекста и качественным instruction-following.

*Модели идентичны PlannerAgent и CriticAgent.*

---

### ReporterAgent — крупная модель (генерация текста)

**Задача**: написание итогового отчёта `report/models.md` на основе реальных данных эксперимента. Требуется связный русскоязычный текст с таблицами и конкретными числами.

*Модели идентичны PlannerAgent.* При недоступности LLM — автоматически генерируется rule-based отчёт из данных эксперимента.

---

## 4. Основной провайдер: HuggingFace Serverless Inference

Система разрабатывалась и тестировалась преимущественно на **HuggingFace Serverless Inference API** (`HF_TOKEN`). Именно этот провайдер использовался как основной в ходе экспериментов.

### Почему HuggingFace

1. **Полностью бесплатно** — Serverless Inference API для публичных моделей не требует оплаты и доступен по токену `HF_TOKEN`, который бесплатно выдаётся при регистрации.
2. **Прямой доступ к открытым весам** — модели запускаются на инфраструктуре HuggingFace без посредников; это гарантирует стабильное поведение и воспроизводимость результатов между запусками.
3. **Богатый выбор моделей** — доступны Qwen 2.5 72B, Llama 3.1 8B, Mistral и другие, охватывающие весь спектр от быстрых tool-calling агентов до reasoning-задач.
4. **Открытые лицензии** — Llama 3.1, Qwen 2.5 имеют пермиссивные лицензии; при необходимости систему можно перевести на локальный деплой (Ollama, vLLM) без изменения кода агентов.
5. **Нет зависимости от проприетарного API** — в отличие от OpenAI / Anthropic, HuggingFace не блокирует доступ по географическому признаку и не требует платёжных данных для старта.

### Роль остальных провайдеров

| Провайдер | Роль в проекте |
|-----------|---------------|
| **HuggingFace** | Основной — все прогоны и эксперименты |
| **OpenRouter** | Резервный при rate-limit HF; удобен для Llama/Mistral через OpenAI-совместимый API |
| **Anthropic** | Эталонное качество для финальной проверки (платно) |
| **VseGPT** | Доступ к o3-mini для сложных reasoning-задач |

### Ограничения HuggingFace Serverless и как система их обходит

- **Нестабильная latency** — тяжёлые агенты (Planner, Critic) назначены на Qwen 2.5 72B; лёгкие (Validator, Explorer, Engineer, Builder) — на Llama 3.1 8B. Это снижает среднее время ожидания.
- **Нет нативного tool use** — бэкенд `hf_backend.py` парсит JSON из текстового ответа, эмулируя function calling. Все агенты работают в этом режиме без изменений.
- **Rate limits** — при 429 система делает экспоненциальный backoff (до 30 с) через базовый retry-цикл в `BaseAgent.run()`.

## 5. Команды для запуска

```bash
# Anthropic Claude (платно, лучшее качество)
$env:ANTHROPIC_API_KEY="sk-ant-..."
python main.py
```

```bash
# OpenRouter (бесплатные open-source модели)
$env:OPENROUTER_API_KEY="sk-or-..."
python main.py
```

```bash
# VseGPT (российский прокси)
$env:VSEGPT_API_KEY="..."
python main.py
```

```bash
# HuggingFace Serverless Inference
$env:HF_TOKEN="hf_..."
python main.py
```

```bash
# Без LLM (rule-based режим: модели обучаются, submission генерируется)
python main.py --no-llm
```

> Провайдер определяется автоматически по наличию ключа в окружении.
> Явно задать провайдер: `$env:LLM_PROVIDER="openrouter"`.
